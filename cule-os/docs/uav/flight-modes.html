Flight Modes

Complete guide to all flight modes in Cule OS UAV Edition

Overview#

Cule OS supports multiple flight modes that provide varying levels of automation. Choose the right mode based on your mission requirements and pilot skill level.

Mode Type Best For
MANUAL Manual Emergency situations
STABILIZE Semi-Auto Learning, manual flight
ALT_HOLD Semi-Auto Photography, inspection
LOITER Semi-Auto Waiting, observation
AUTO Autonomous Missions, waypoints
RTL Autonomous Emergency return
GUIDED Autonomous Ground control

Manual Modes#

MANUAL Manual#

Direct RC control with no stabilization. The flight controller passes through RC signals directly to motors/servos.

Use When:

  • Emergency situations requiring direct control
  • Testing hardware without flight controller assistance

Requirements: RC transmitter properly configured

Semi-Automatic Modes#

STABILIZE Semi-Auto#

Self-leveling flight mode. The flight controller actively corrects to level when you release the sticks, but throttle is manual.

Use When:

  • Learning to fly
  • Manual flight with stability assistance
cule-mode stabilize

ALT_HOLD Semi-Auto#

Maintains altitude automatically using barometer. You control pitch, roll, and yaw while the flight controller manages throttle to maintain altitude.

Use When:

  • Aerial photography
  • Inspection missions
  • Smooth video capture

Requirements: Barometer calibrated, GPS lock recommended

LOITER Semi-Auto#

Holds position and altitude using GPS. The drone maintains its current location and altitude automatically.

Use When:

  • Waiting for commands
  • Observation
  • Setting up for next action

Requirements: 3D GPS lock (HDOP < 2.0)

Autonomous Modes#

AUTO Autonomous#

Executes pre-programmed mission waypoints. The drone flies autonomously following the loaded mission.

Use When:

  • Survey missions
  • Mapping
  • Delivery routes
# Load and execute mission
cule-mission load survey.plan
cule-arm --mode auto

# Monitor mission progress
cule-status mission

RTL (Return to Launch) Autonomous#

Returns to the takeoff location and lands automatically. Used for failsafe scenarios.

Use When:

  • Low battery
  • Signal loss (failsafe)
  • End of mission
⚠️ Important: RTL altitude must be set higher than any obstacles between current position and home.

GUIDED Autonomous#

Ground station controlled mode. Accepts position and velocity commands from companion computer or ground station.

Use When:

  • Follow-me mode
  • Point-and-click navigation
  • Computer vision guidance

Failsafe Behavior#

When failsafe triggers (RC loss, low battery, etc.), the drone automatically switches to RTL mode. Configure in /etc/cule/failsafe.conf:

# /etc/cule/failsafe.conf
FS_THR_ENABLE=1          # Enable throttle failsafe
FS_BATT_ENABLE=1         # Enable battery failsafe
FS_GCS_ENABLE=1          # Enable GCS failsafe
RTL_ALT=3000             # RTL altitude in cm (30m)
LAND_SPEED=50            # Landing speed cm/s
Learn More: See Mission Planning for creating autonomous flight plans.
